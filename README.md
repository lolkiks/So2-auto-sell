# So2-auto-sell

Скрипт для **Perfect Click** (упрощённый синтаксис, без объектов),
в котором можно отдельно задать:
- цену продажи,
- координаты кнопок,
- **координаты поля ввода цены**.

```js
// ===== НАСТРОЙКИ =====
var PRICE = "50.00"; // цена продажи

// Координаты кнопок/элементов (подставь свои)
var sellX = 540;
var sellY = 1765;

var skinX = 540;
var skinY = 620;

// Координаты поля ВВОДА ЦЕНЫ
var priceFieldX = 540;
var priceFieldY = 1450;

var okX = 740;
var okY = 1700;

var createX = 540;
var createY = 1850;

// ===== СЦЕНАРИЙ =====
sleep(2000); // время переключиться в игру

click(sellX, sellY);
sleep(1000);

click(skinX, skinY);
sleep(1000);

// Клик по полю ввода цены
click(priceFieldX, priceFieldY);
sleep(600);

// Ввод цены (если функция поддерживается твоей версией)
if (typeof setText === "function") {
  setText(PRICE);
  sleep(500);
}

click(okX, okY);
sleep(900);

click(createX, createY);
sleep(900);
```

Если у тебя в Perfect Click не работает `setText`, оставь только клик по `priceFieldX/priceFieldY` и вводи цену вручную с клавиатуры.

## Новый универсальный промт (без нарушения правил игр)

Ниже — безопасный промт для генерации Java-скрипта под **автоматизацию тестового Android-приложения** в эмуляторе (а не для обхода античита или автоматизации торговли в онлайн-играх):

```text
Действуй как Senior Java Automation Engineer.

Нужен каркас Java 17+ проекта для автоматизации действий в Android-эмуляторе в тестовом приложении MarketplaceDemo.

Цель:
- мониторить экран,
- находить элементы интерфейса по шаблонам,
- считывать цену из заданной зоны,
- выполнять покупку только если price <= targetPrice,
- логировать все шаги и корректно обрабатывать ошибки.

Требования:
1) Архитектура:
   - Раздели код на классы:
     - ScreenCaptureService (ADB screencap + загрузка bitmap)
     - VisionService (OpenCV template matching)
     - OcrService (Tesseract OCR)
     - InputService (ADB tap)
     - TradeEngine (бизнес-логика)
     - Config (координаты, пороги, тайминги)
   - Используй конфиг-файл application.yaml.

2) Технологии:
   - Java 17
   - OpenCV
   - Tess4J (Tesseract)
   - slf4j + logback
   - Maven

3) Логика:
   - Цикл мониторинга каждые pollIntervalMs (по умолчанию 100 мс).
   - Если найден элемент buyButton и цена распознана:
       IF price <= targetPrice THEN
           - Проверить доступный баланс
           - Нажать "Купить"
           - Проверить окно подтверждения
           - Нажать "Подтвердить"
       ELSE
           - Лог "цена выше порога"
   - Если любой шаг неуспешен, вернуть систему в состояние ожидания.

4) Надежность:
   - Retry для ADB-команд (до 3 попыток).
   - Таймауты на каждый этап.
   - Обработка исключений без падения всего цикла.
   - Скриншот при ошибке в папку /logs/screenshots.

5) Вывод:
   - Дай полный минимально-рабочий код (MVP):
     - pom.xml
     - Main.java
     - все классы сервисов
     - пример application.yaml
   - Добавь комментарии на русском языке.
   - В конце добавь блок "Как запустить" с командами Maven.
```
